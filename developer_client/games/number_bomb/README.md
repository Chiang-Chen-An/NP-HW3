# 💣 Number Bomb Game

## 遊戲簡介
數字炸彈是一個刺激的多人策略遊戲。玩家輪流在當前數字上加 1、2 或 3，誰讓數字達到炸彈數字就輸了！

## 遊戲規則

### 基本規則
- **玩家人數**: 2-4 人
- **炸彈數字**: 隨機設定在 15-30 之間
- **起始數字**: 0
- **每回合選擇**: 玩家可以選擇加 1、2 或 3

### 遊戲流程
1. 遊戲開始時，系統隨機設定一個炸彈數字（15-30）
2. 從隨機玩家開始，依序輪流
3. 每個玩家在自己的回合必須選擇加 1、2 或 3
4. 當前數字會累加選擇的數字
5. 如果玩家讓數字剛好達到炸彈數字，該玩家出局
6. 出局後，數字重置為 0，剩餘玩家繼續遊戲
7. 當只剩一個玩家時，該玩家獲勝

### 策略提示
- 🧠 計算距離炸彈數字的距離
- 🎯 嘗試讓對手陷入必輸的局面
- ⚠️ 當數字接近炸彈時要特別小心
- 💡 有時候加 1 比加 3 更安全

## GUI 功能

### 主要界面元素
1. **當前數字顯示區** (大字)
   - 顯示當前累計的數字
   - 顏色會根據危險程度變化：
     - 🔵 藍色：安全 (距離炸彈 > 6)
     - 🟠 橙色：警告 (距離炸彈 4-6)
     - 🔴 紅色：危險 (距離炸彈 ≤ 3)

2. **炸彈數字顯示**
   - 顯示目標炸彈數字

3. **玩家列表**
   - 顯示所有玩家狀態
   - 👉 YOUR TURN：輪到該玩家
   - ⏳ Waiting：等待中
   - 💀 OUT：已出局

4. **操作按鈕**
   - 🟢 +1 按鈕（綠色）
   - 🟡 +2 按鈕（橙色）
   - 🔴 +3 按鈕（紅色）
   - 按鈕會根據剩餘空間自動禁用

5. **遊戲歷史記錄**
   - 顯示所有玩家的操作歷史
   - 包含時間戳記
   - 自動滾動顯示最新記錄

### 視覺特效
- 💥 爆炸效果（踩到炸彈時）
- 🎨 顏色警告系統
- 🎉 勝利動畫
- ⏰ 時間戳記歷史記錄

## 遊戲範例

### 範例局面 (炸彈數字 = 21)
```
回合 1: Player1 加 3 → 數字變成 3
回合 2: Player2 加 2 → 數字變成 5
回合 3: Player3 加 3 → 數字變成 8
回合 4: Player4 加 1 → 數字變成 9
回合 5: Player1 加 3 → 數字變成 12
回合 6: Player2 加 3 → 數字變成 15
回合 7: Player3 加 3 → 數字變成 18 (危險！)
回合 8: Player4 必須選擇...
  - 加 1 → 19 (還安全)
  - 加 2 → 20 (很危險！)
  - 加 3 → 21 (💥 出局！)
```

### 策略分析
在上面的例子中，Player4 應該加 1，因為：
- 加 1 → 19，下一個玩家只能加 1 或 2
- 如果下一個玩家加 1 → 20，那麼 Player4 就能加 1 讓對手陷入必輸局面

## 技術規格

### 伺服器
- 支援 2-4 個玩家同時遊戲
- 使用 Socket 通訊
- JSON 格式的訊息封包
- 執行緒安全的狀態管理

### 客戶端
- Tkinter GUI 界面
- 非同步訊息接收
- 即時遊戲狀態更新
- 使用者友善的操作介面

### 訊息格式
```python
# INIT - 初始化玩家
{
    "type": "INIT",
    "data": {"player_id": "Player1"}
}

# UPDATE - 遊戲狀態更新
{
    "type": "UPDATE",
    "data": {
        "current_number": 15,
        "bomb_number": 21,
        "current_turn": "Player2",
        "players": ["Player1", "Player2", "Player3", "Player4"],
        "eliminated": ["Player3"],
        "last_move": {
            "player": "Player1",
            "added": 2,
            "old_number": 13,
            "new_number": 15
        }
    }
}

# INPUT - 玩家輸入
{
    "type": "INPUT",
    "data": {"move": 2}
}

# RESULT - 遊戲結果
{
    "type": "RESULT",
    "data": {
        "winner": "Player1",
        "loser": "Player2",
        "final_number": 21
    }
}
```
